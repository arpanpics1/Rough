1. Trigger Copilot Suggestions
Tab: Accept the current suggestion.
Esc: Dismiss the current suggestion.
Ctrl + Enter (Windows/Linux) or Cmd + Enter (Mac): Open the Copilot feedback panel to view more options or get additional suggestions.
Alt + ] (Windows/Linux) or Option + ] (Mac): View the next Copilot suggestion.
2. Manual Invocation of Copilot
Type a Comment: Start by writing a descriptive comment (e.g., # create a function to calculate Fibonacci). Copilot often generates code based on the comment.
Use Shortcut for Inline Suggestions: Ctrl + Space (for VS Code) or Alt + \ (Windows/Linux) or Option + \ (Mac) to open the suggestion menu manually if Copilot doesn't automatically provide a suggestion.
3. Code Blocks and Examples
When working on larger code structures, write comments or code blocks as context for Copilot, allowing it to understand your code better and generate relevant code snippets.
4. Editing and Refining Suggestions
Highlight a Code Block + Trigger Copilot: Highlight existing code and ask Copilot to refactor it by triggering suggestions again. This can provide different versions of the code for optimization or readability.
5. Documentation Generation
Write a comment like # generate documentation for this function above a function or class to let Copilot auto-generate documentation.
6. Ask for Code Explanations
Comment on a complex piece of code (e.g., # explain this function) to get an automatic explanation from Copilot, which can help you understand complex code.
7. Request Tests for Functions
Write a comment like # generate unit tests for this function above a function to let Copilot create test cases.
8. Refactoring Suggestions
Comments like # optimize this loop or # improve performance of this function before a code block can prompt Copilot to refactor for performance.
9. Code Format and Linting
If you're using Prettier or another formatter, Copilot will generally format based on those rules. Adding comments for specific formatting styles can help guide Copilot’s formatting style.
